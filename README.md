# AI Interview Assistant

Personalised mock interviews powered by an OpenAI-compatible LLM (works with OpenAI, Groq, DeepSeek, etc.).

## Architecture

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│  Chainlit   │────▶│   app.py     │────▶│ llm_client   │
│  (UI)       │◀────│  (controller)│◀────│  (LLM calls) │
└─────────────┘     └──────────────┘     └──────┬───────┘
                                                │
                    ┌───────────────────────────┼───────────┐
                    │                           │           │
            ┌───────▼───────┐         ┌─────────▼───────┐   │
            │  session_state│         │  prompts.py     │   │
            │  (state mach.)│         │  (templates)    │   │
            └───────────────┘         └─────────────────┘   │
                    │                                        │
            ┌───────▼───────┐         ┌─────────────────┐   │
            │  scoring.py   │         │  fallback_data  │   │
            │  (weights,    │         │  (static Qs)    │   │
            │   radar chart)│         └─────────────────┘   │
            └───────────────┘                                │
                    │                                        │
            ┌───────▼───────┐         ┌─────────────────┐   │
            │  export.py    │         │  timer.py       │   │
            │  (MD + PDF)   │         │  (timeout)      │   │
            └───────────────┘         └─────────────────┘   │
                    └───────────────────────────────────────┘
```

### State Machine

```
IDLE → ONBOARDING → GENERATING → INTERVIEWING → EVALUATING → FEEDBACK → COMPLETED → DEBRIEF
```

## Setup

```bash
cp .env.example .env
# Fill in your API key in .env
uv sync
```

### Configuration (`.env`)

| Variable | Default | Description |
|----------|---------|-------------|
| `LLM_PROVIDER` | `openai` | Only `openai` is supported |
| `OPENAI_API_KEY` | — | Your API key (OpenAI, Groq, DeepSeek, etc.) |
| `OPENAI_BASE_URL` | — | Custom endpoint (e.g. `https://api.groq.com/openai/v1`) |
| `OPENAI_MODEL` | `gpt-4o-mini` | Model name |
| `QUESTION_TIMER_SECONDS` | `180` | Answer timeout in seconds |

### Provider Switching

This app uses the OpenAI SDK, which is compatible with any provider that offers an OpenAI-compatible API.

**Groq:**
```env
OPENAI_BASE_URL=https://api.groq.com/openai/v1
OPENAI_MODEL=llama-3.3-70b-versatile
```

**DeepSeek:**
```env
OPENAI_BASE_URL=https://api.deepseek.com
OPENAI_MODEL=deepseek-chat
```

**OpenAI:**
```env
# No base URL needed for default OpenAI
OPENAI_MODEL=gpt-4o-mini
```

## Features

- **Personalised questions** — generated by LLM based on role, seniority, and industry
- **Multi-dimension scoring** — clarity, completeness, relevance, grammar, impact (weighted)
- **Radar chart** — visual breakdown of performance across dimensions
- **Anti-injection** — guards against prompt injection in answers
- **Fallback questions** — static question bank when API is unavailable
- **Retry logic** — auto-retries on API failures with exponential backoff
- **Timer enforcement** — auto-skips unanswered questions after timeout
- **PDF/Markdown export** — download full interview transcript and scorecard

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `API key not configured` | Set `OPENAI_API_KEY` in `.env` |
| `LLM call failed after 2 retries` | Check API key, network, or provider status |
| `No valid transition` | You hit an unexpected state; type `start` to restart |
| Evaluation stuck on loading | Click **Retry Evaluation** or skip to next question |
| Timer expired unexpectedly | Adjust `QUESTION_TIMER_SECONDS` in `.env` |
| WeasyPrint PDF error | Ensure system deps: `libpango-1.0-0`, `libcairo2` |
